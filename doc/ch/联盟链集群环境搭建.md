# 联盟链开发环境搭建
联盟链环境搭建跟私有链类似，可通过多服务器部署或docker部署。chain33联盟链采用自研的qbft共识，节点数要满足N>3f，至少需要3f+1个节点。  

## 多服务器部署

### 环境准备
- 下载软件包（在其中一台操作）
```bash
wget https://bty33.oss-cn-shanghai.aliyuncs.com/chain33Dev/chain33_Consortium_1.65.2.tar.gz
```

- 下载后解压到任意指定目录
```bash
tar -xzvf chain33_Consortium_1.65.2.tar.gz
```

- 软件包包含
```text
chain33         -- chain33节点程序
chain33-cli     -- chain33节点命令行工具
chain33.toml    -- chain33主链配置文件
genconfig.sh    -- 节点目录生成工具
```

### 生成节点目录
假设4台服务器的ip分别为`172.27.0.2`, `172.27.0.5`, `172.27.0.9`, `172.27.0.15`   （在云服务器环境下，IP使用内网地址，千兆网络P2P传输广播效率高）
```bash
  # 赋权限
  chmod u+x chain33 chain33-cli chain33.toml 
  # 生成四个节点的区块链程序及配置文件
./genconfig.sh "172.27.0.2,172.27.0.5,172.27.0.9,172.27.0.15"
```

生成4个名字为ip的目录，将这4个目录（172.27.0.2,172.27.0.5,172.27.0.9,172.27.0.15）分别发送到各个服务器上（目录可以自己重命名）。
注意： 4台服务器之间需要开通13802（p2p端口）和33001(共识端口)，以及8801(区块链对外jsonrpc端口)

### 启动节点

登录到各个节点执行启动命令，顺序不分前后，建议在一分钟内全都启动

```bash
nohup ./chain33 -f chain33.toml >/dev/null 2>&1 &
```

查看进程

```bash
ps -ef | grep -v grep | grep chain33
```

如果进程已经启动，执行命令查询节点同步状态(5分钟左右)

```bash
./chain33-cli net is_sync
```

返回结果为true则说明运行成功。

执行命令检查p2p节点信息
```bash
./chain33-cli net peer
```

## docker部署（只用于开发调试）

### 环境准备
- 下载软件包
```bash
wget https://bty33.oss-cn-shanghai.aliyuncs.com/chain33Dev/mainet/tendermint/chain33.tendermint.docker.tar.gz
```

- 下载后解压到任意指定目录
```bash
mkdir chain33
tar -xzvf chain33.tendermint.docker.tar.gz -C chain33
```

- 软件包包含
```text
chain33            -- chain33节点程序
chain33-cli        -- chain33节点命令行工具
chain33.toml       -- chain33主链配置文件
docker-build.sh    -- docker镜像生成脚本
docker-install.sh  -- docker命令安装降本
README.md          -- 使用说明
```

### 启动

#### 安装docker
如果服务器上没有安装docker命令，可使用docker-install.sh安装，如果已经安装则跳过
```bash
./docker-install.sh
```

#### 生成配置文件
docker-build.sh脚本用于生成chain33镜像和对应的配置文件
```bash
./docker-build.sh
```
默认使用4个节点和172.50.0.0网段，用户也可以自己配置
```bash
Usage:
./docker-build.sh <num> <network>
    num --- 节点数量，默认是4个节点
    network -- 节点网络，默认是172.50.0.0

eg.
./docker-build.sh 7 172.10.0.0
```

执行成功后会生成`docker-compose.yaml`和`Dockerfile`，以及对应节点的挂载目录

#### 启动docker
```bash
docker-compose -f docker-compose.yaml up --build -d
```

## 添加删除节点

### 添加共识节点


#### 0 保证已有添加或者删除节点权限的账户qbft-manager，如果没有，在原有节点上执行以下命令设置
```bash
# 配置qbft-manager，-c：qbft-manager，-o：add(表示添加)，-v：qbft-manager的地址，-k：区块链的超级管理员的私钥
$ ./chain33-cli config config_tx -c qbft-manager -o add -v 1CbEVT9RnM5oZhWMj4fxUrJX94VtRotzvs
# 查询结果
$ ./chain33-cli config query -k qbft-manager
# 输出样例
{
"key": "qbft-manager",
 "value": "[1CbEVT9RnM5oZhWMj4fxUrJX94VtRotzvs]"
}
```
#### 1 新节点配置公私文件，私钥文件，chain33，chain33-cli，chain33.toml
```bash
# 生成私钥文件，-n：生成数量，-t：签名算法，默认生成私钥文件priv_validator_0.json
$ ./chain33-cli qbft gen_file -n 1 -t bls
```
将priv_validator_0.json重命名为共识配置中私钥文件名称priv_validator.json
公钥文件genesis.json，chain33，chain33-cli，chain33.toml从原有机器上拷贝

#### 2 新节点启动chain33，等到同步到最新高度区块

#### 3 添加共识节点，假设节点私钥为F4E29F6C49D4BE13A9271C0E66D96413C64A0D6C
```bash
# 添加共识节点，-p：新节点的私钥，-w：节点权重，不超过原有节点权重之和的 1/3，-k：qbft-manager账户私钥
./chain33-cli send qbft add -p F4E29F6C49D4BE13A9271C0E66D96413C64A0D6C -w 10 -k 3990969DF92A5914F7B71EEB9A4E58D6E255F32BF042FEA5318FC8B3D50EE6E8
```


### 删除共识节点

```bash
# 删除共识节点，-p：删除节点的私钥，-w：节点权重，0表示删除，-k：qbft-manager账户私钥
./chain33-cli send qbft add -p F4E29F6C49D4BE13A9271C0E66D96413C64A0D6C -w 0 -k 3990969DF92A5914F7B71EEB9A4E58D6E255F32BF042FEA5318FC8B3D50EE6E8
```

执行 “添加共识节点” 步骤1，然后启动chain33
